# Alpha Core Memory

This file stores durable team/project memory: decisions, principles, constraints, and lessons.

## Core Thoughts (Initial)
- Keep the team structure lean and domain-complete (7 personas).
- Favor clarity, traceability, and maintainability over speed theater.
- Every sprint should produce shippable value and observable outcomes.
- Handoffs must be explicit, timestamped, and include the next prescribed step.
- Every dev round must include per-discipline voice logging (including objections/advice/guidance/admonishment/frustration).
- Execute independently by default; escalate before likely failure.
- PRD is intentionally resettable each dev round.
